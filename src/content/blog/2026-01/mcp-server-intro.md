# MCP(Model Context Protocol) 서버란?

MCP(Model Context Protocol)는 AI 언어 모델과 외부 데이터 소스, 도구, 시스템을 원활하게 연결하는 **오픈 표준 프로토콜**입니다.

## 왜 MCP가 필요한가?

AI 시스템은 오랫동안 데이터 저장소, 비즈니스 도구, 레거시 시스템과 분리되어 있었습니다. MCP는 이러한 **정보 사일로**를 해결하고, AI가 실제 데이터가 있는 곳에 연결될 수 있도록 합니다.

개발자는 각 데이터 소스마다 별도의 커넥터를 유지할 필요 없이, **단일 표준 프로토콜**만으로 다양한 시스템과 연동할 수 있습니다.

## MCP 서버의 3가지 핵심 기능

MCP 서버는 언어 모델에 컨텍스트를 추가하기 위해 세 가지 기본 기능을 제공합니다.

| 기능 | 제어 주체 | 설명 | 예시 |
|------|----------|------|------|
| **Prompts** | 사용자 | 사용자 선택으로 호출되는 대화형 템플릿 | 슬래시 명령어, 메뉴 옵션 |
| **Resources** | 애플리케이션 | 클라이언트가 관리하는 컨텍스트 데이터 | 파일 내용, Git 히스토리 |
| **Tools** | 모델 | LLM이 실제 작업을 수행할 수 있는 실행 함수 | API 요청, 파일 쓰기 |

### Prompts (프롬프트)

사용자가 직접 호출하는 미리 정의된 템플릿입니다. 슬래시 명령어나 메뉴를 통해 원하는 지시를 선택할 수 있습니다.

### Resources (리소스)

파일, Git 히스토리처럼 **애플리케이션이 관리하는 컨텍스트 데이터**입니다. 클라이언트가 필요한 리소스를 모델에 첨부하여 더 정확한 응답을 유도합니다.

### Tools (도구)

LLM이 **직접 액션을 취할 수 있는 실행 가능한 함수**입니다. API 호출, 파일 작성, 데이터베이스 조회 등 실제 작업을 수행합니다.

## 활용 사례

MCP 호환 서버는 이미 다양한 기업 시스템과 연동되고 있습니다.

- **Google Drive, Slack, GitHub, Git** — 협업 및 개발 도구
- **Postgres** — 데이터베이스 쿼리
- **Puppeteer** — 브라우저 자동화

또한 Zed, Replit, Codeium, Sourcegraph 같은 개발 도구들이 MCP를 채택하고 있으며, 766개 이상의 MCP 호환 서버가 문서화되어 있습니다.

## 요약

MCP는 AI와 실제 세계의 데이터·도구를 연결하는 **범용 프로토콜**입니다. Prompts, Resources, Tools 세 가지 기본 요소를 통해 AI 애플리케이션이 더 풍부하고 실용적인 컨텍스트를 활용할 수 있게 합니다.
